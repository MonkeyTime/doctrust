# DocTrust 0.2.0

This release adds first-class payment profile validation to the reference SDKs.

## Included

- `validatePaymentProfile(...)` in JavaScript, Python, and .NET
- spec and schema updates for the payment profile
- contribution and security issue templates
- updated demo flow that validates a payment profile

## Demo

Run:

```bash
cd sdk-js
node ../demo/repro.mjs
```

Expected output includes:

- a verified payload
- `transportPrefix` equal to `dtp1z.`
- `paymentProfileOk` set to `true`
- `tamperRejected` set to `true`
- payment profile validation passing against the expected transfer details

## Notes

This release keeps the project focused on invoice and payment instruction verification.
It also tightens the reference implementations so payment validation requires an explicit transfer context, trust registry entries must bind issuer IDs explicitly, and compact QR payloads are capped against decompression bombs.
